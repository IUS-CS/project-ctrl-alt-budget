# Testing Methodology

We are currently using unit testing to ensure part of our project are working correctly. We make use of pytest for our unit testing and in doing so we ensure our backend is well established. For future merges and program testing we will do integration testing to make sure no merges interfere with current working units. 

## Unit Testing

For our unit testing, we make use of PyTest to write testing file and run multiple tests at once. Using PyTest helps us ensure that our testing covers certain areas and helps identify specific areas where code my be buggy. We store all our tests in their own directory under src/test. We attempt to cover all edge cases and potential unexpected errors when testing our units.
